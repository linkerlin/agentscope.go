## 1. PersonalSummarizer (个人记忆提取器)

### 1.1 功能概述

从用户对话中自动提取持久性个人信息，包括偏好、习惯、背景等。

### 1.2 核心方法

```go
// 创建提取器
summarizer := NewPersonalSummarizer(model, "zh")

// 提取观察记忆
observations, err := summarizer.ExtractObservations(ctx, messages, "user_alice")

// 提取洞察（更高层次的总结）
insights, err := summarizer.ExtractInsights(ctx, observations, "user_alice")

// 基于新观察更新已有洞察
updated, err := summarizer.UpdateInsights(ctx, insights, newObservations, "user_alice")

// 处理矛盾/重复记忆
filtered, deletedIDs, err := summarizer.HandleContraRepeat(ctx, memories)
```

### 1.3 处理流程

```
原始对话
    ↓
[时间相关过滤] ← 过滤包含"今天"、"明天"等时间词的消息
    ↓
[观察提取] ← LLM提取事实性信息
    ↓
[矛盾/重复检测] ← 识别并移除矛盾或重复的记忆
    ↓
[洞察生成] ← 从观察中总结深层洞察
    ↓
[洞察更新] ← 基于新观察更新已有洞察
    ↓
存储到向量库
```

### 1.4 提示词工程

**观察提取提示词**:
```
从以下对话中提取关于用户的持久性信息（偏好、习惯、背景等）。

规则:
1. 只提取事实性信息，不猜测
2. 提取的信息应该是长期有效的（非时间相关的）
3. 格式: 信息：<序号> <> <内容> <关键词>
4. 如果没有有效信息，输出: 无

示例:
信息：<1> <> <用户喜欢喝咖啡> <饮食偏好>
信息：<2> <> <用户是软件工程师> <职业>
```

**矛盾/重复检测提示词**:
```
分析以下记忆列表，识别矛盾或重复的记忆。

规则:
- 矛盾：两个记忆内容相互矛盾
- 被包含：一个记忆的内容被另一个完全包含
- 无：记忆有效且独立

格式: <序号> <矛盾|被包含|无>
```

## 2. ProceduralSummarizer (程序记忆提取器)

### 2.1 功能概述

从任务执行轨迹中提取可复用的经验，包括成功模式、失败教训、改进建议。

### 2.2 核心方法

```go
// 创建提取器
summarizer := NewProceduralSummarizer(model, "zh")

// 从多条轨迹提取
memories, err := summarizer.ExtractFromTrajectories(ctx, trajectories)

// 从单条轨迹提取
memories, err := summarizer.ExtractFromSingleTrajectory(ctx, trajectory)

// 对比学习：从多条成功轨迹提取共同模式
patterns, err := summarizer.ExtractSuccessPattern(ctx, successfulTrajectories)

// 从失败轨迹提取教训
lessons, err := summarizer.ExtractFailureLesson(ctx, failedTrajectories)

// 验证记忆质量
valid, invalid := summarizer.ValidateMemories(ctx, memories, sampleTrajectory)
```

### 2.3 轨迹评分策略

| 评分 | 分类 | 处理策略 |
|-----|------|---------|
| ≥ 0.9 | 成功 | 提取成功模式、最佳实践 |
| 0.5 - 0.9 | 部分成功 | 提取改进建议 |
| < 0.5 | 失败 | 提取失败教训、避免方法 |

### 2.4 提取示例

**输入轨迹**:
```
User: 帮我优化这个数据库查询
Assistant: [调用 explain_query 工具]
Tool: 查询使用了全表扫描，耗时 3.2s
Assistant: [调用 add_index 工具]
Tool: 索引添加成功
Assistant: [调用 explain_query 工具]
Tool: 查询使用索引，耗时 0.05s
```

**提取的记忆**:
```json
{
  "when_to_use": "处理数据库查询优化时",
  "memory": "优先检查是否使用索引，避免全表扫描。可使用 EXPLAIN 分析查询计划。",
  "metadata": {
    "pattern_type": "success",
    "source_score": 1.0
  }
}
```

## 3. ToolSummarizer (工具记忆提取器)

### 3.1 功能概述

记录工具调用历史，评估调用质量，生成动态使用指南。

### 3.2 数据结构

```go
type ToolCallResult struct {
    CreateTime   time.Time
    ToolName     string
    Input        map[string]any
    Output       string
    TokenCost    int
    Success      bool
    TimeCost     float64
    IsSummarized bool
    Summary      string  // LLM生成的调用摘要
    Evaluation   string  // LLM对调用质量的评价
    Score        float64 // 评分 0.0-1.0
}
```

### 3.3 核心方法

```go
// 创建提取器
summarizer := NewToolSummarizer(model, "zh")

// 评估单次调用
err := summarizer.EvaluateToolCall(ctx, &result)

// 批量评估
err := summarizer.BatchEvaluate(ctx, results)

// 生成使用指南
guide, err := summarizer.SummarizeToolUsage(ctx, "web_search", results)

// 生成最佳实践文档
bestPractices, err := summarizer.GenerateBestPractices(ctx, "web_search", successfulResults)
```

### 3.4 评估维度

```
ToolCallResult
├── 执行维度
│   ├── Success: 是否成功执行
│   ├── TimeCost: 执行耗时
│   └── TokenCost: Token消耗
│
├── 质量维度 (LLM评估)
│   ├── Summary: 调用目的和结果摘要
│   ├── Evaluation: 参数选择、结果质量评价
│   └── Score: 综合评分 (0.0-1.0)
│
└── 状态
    └── IsSummarized: 是否已被总结
```

### 3.5 使用指南生成

基于最近的调用记录（默认30条），生成包含以下内容的使用指南：

1. **最佳使用场景**: 什么情况下使用该工具效果最好
2. **参数优化建议**: 如何设置参数以获得最佳结果
3. **常见错误**: 常见错误和避免方法
4. **性能优化**: 提升执行效率的技巧
5. **统计信息**: 成功率、平均耗时、Token消耗

## 4. MemoryDeduplicator (记忆去重器)

### 4.1 功能概述

多层次的记忆去重，包括向量相似度去重、LLM语义去重、矛盾检测。

### 4.2 去重策略

```
记忆列表
    ↓
[向量相似度去重] ← 基于嵌入向量的余弦相似度
    ↓
[LLM语义去重] ← 使用LLM判断语义重复 (可选)
    ↓
[矛盾检测] ← 识别相互矛盾的记忆
    ↓
[相似记忆合并] ← 合并高度相似的记忆
    ↓
去重后的记忆列表
```

### 4.3 核心方法

```go
// 创建去重器
deduplicator := NewMemoryDeduplicator(embeddingModel).WithLLM(model)

// 基本去重
unique, removedIDs, err := deduplicator.Deduplicate(ctx, memories)

// 与存储去重（防止重复添加到库中）
unique, err := deduplicator.DeduplicateAgainstStore(ctx, newMemories, vectorStore)

// 矛盾检测
contradictions, err := deduplicator.FindContradictions(ctx, memories)
// 返回: [[记忆A, 记忆B], [记忆C, 记忆D]] 表示A与B矛盾，C与D矛盾

// 合并相似记忆
merged, err := deduplicator.MergeSimilarMemories(ctx, memories)

// 简单去重（无需嵌入模型）
unique, removed := SimpleDeduplicate(memories, 0.85)
```

### 4.4 相似度阈值

| 阈值 | 用途 | 说明 |
|-----|------|------|
| 0.95+ | 完全重复 | 几乎相同的内容 |
| 0.85-0.95 | 高度相似 | 可以合并 |
| 0.70-0.85 | 相似 | 保留，但标记为相关 |
| < 0.70 | 不同 | 独立记忆 |

## 5. 整合使用示例

### 5.1 完整的个人记忆流程

```go
func PersonalMemoryWorkflow(ctx context.Context, vectorMem *ReMeVectorMemory, model model.ChatModel, msgs []*message.Msg, userName string) error {
    // 1. 创建提取器
    summarizer := NewPersonalSummarizer(model, "zh")
    
    // 2. 提取观察
    observations, err := summarizer.ExtractObservations(ctx, msgs, userName)
    if err != nil {
        return err
    }
    
    // 3. 处理矛盾/重复
    filtered, _, err := summarizer.HandleContraRepeat(ctx, observations)
    if err != nil {
        return err
    }
    
    // 4. 提取洞察
    insights, err := summarizer.ExtractInsights(ctx, filtered, userName)
    if err != nil {
        return err
    }
    
    // 5. 合并观察和洞察
    allMemories := append(filtered, insights...)
    
    // 6. 去重
    deduplicator := NewMemoryDeduplicator(vectorMem.store.embed)
    unique, _, err := deduplicator.Deduplicate(ctx, allMemories)
    if err != nil {
        return err
    }
    
    // 7. 存储到向量库
    for _, mem := range unique {
        if err := vectorMem.AddMemory(ctx, mem); err != nil {
            return err
        }
    }
    
    return nil
}
```

### 5.2 完整的任务记忆流程

```go
func TaskMemoryWorkflow(ctx context.Context, vectorMem *ReMeVectorMemory, model model.ChatModel, trajectories []Trajectory) error {
    // 1. 创建提取器
    summarizer := NewProceduralSummarizer(model, "zh")
    
    // 2. 分类轨迹
    var successTraj, failTraj []Trajectory
    for _, t := range trajectories {
        if t.Score >= 0.9 {
            successTraj = append(successTraj, t)
        } else if t.Score < 0.5 {
            failTraj = append(failTraj, t)
        }
    }
    
    var allMemories []*MemoryNode
    
    // 3. 提取成功模式（使用对比学习）
    if len(successTraj) > 0 {
        patterns, err := summarizer.ExtractSuccessPattern(ctx, successTraj)
        if err == nil {
            allMemories = append(allMemories, patterns...)
        }
    }
    
    // 4. 提取失败教训
    if len(failTraj) > 0 {
        lessons, err := summarizer.ExtractFailureLesson(ctx, failTraj)
        if err == nil {
            allMemories = append(allMemories, lessons...)
        }
    }
    
    // 5. 处理中间评分的轨迹
    for _, t := range trajectories {
        if t.Score >= 0.5 && t.Score < 0.9 {
            mems, err := summarizer.ExtractFromSingleTrajectory(ctx, t)
            if err == nil {
                allMemories = append(allMemories, mems...)
            }
        }
    }
    
    // 6. 去重
    deduplicator := NewMemoryDeduplicator(vectorMem.store.embed)
    unique, _, err := deduplicator.Deduplicate(ctx, allMemories)
    if err != nil {
        return err
    }
    
    // 7. 验证记忆质量
    if len(trajectories) > 0 {
        valid, _ := summarizer.ValidateMemories(ctx, unique, trajectories[0])
        unique = valid
    }
    
    // 8. 存储
    for _, mem := range unique {
        if err := vectorMem.AddMemory(ctx, mem); err != nil {
            return err
        }
    }
    
    return nil
}
```

### 5.3 完整的工具记忆流程

```go
func ToolMemoryWorkflow(ctx context.Context, vectorMem *ReMeVectorMemory, model model.ChatModel, toolName string, results []ToolCallResult) error {
    // 1. 创建提取器
    summarizer := NewToolSummarizer(model, "zh")
    
    // 2. 评估未评估的调用
    var toEvaluate []*ToolCallResult
    for i := range results {
        if results[i].Summary == "" {
            toEvaluate = append(toEvaluate, &results[i])
        }
    }
    
    if err := summarizer.BatchEvaluate(ctx, toEvaluate); err != nil {
        return err
    }
    
    // 3. 检查是否需要总结（是否有未总结的记录）
    hasUnsummarized := false
    for _, r := range results {
        if !r.IsSummarized {
            hasUnsummarized = true
            break
        }
    }
    
    if !hasUnsummarized {
        return nil // 没有新记录需要总结
    }
    
    // 4. 生成使用指南
    guide, err := summarizer.SummarizeToolUsage(ctx, toolName, results)
    if err != nil {
        return err
    }
    
    if guide == nil {
        return nil
    }
    
    // 5. 检查是否已有该工具的指南
    existing, err := vectorMem.RetrieveTool(ctx, toolName, toolName, 1)
    if err == nil && len(existing) > 0 {
        // 更新已有指南
        guide.MemoryID = existing[0].MemoryID
        if err := vectorMem.UpdateMemory(ctx, guide); err != nil {
            return err
        }
    } else {
        // 添加新指南
        if err := vectorMem.AddMemory(ctx, guide); err != nil {
            return err
        }
    }
    
    return nil
}
```

> **补充说明**: 当前 ToolMemory 已支持 ReActAgent 自动触发，无需手动调用 Workflow。ReActAgent 在每次工具调用完成后会自动评估调用质量，并在满足条件时自动生成/更新工具使用指南，实现记忆闭环。

### 6.3 缓存策略

`EmbeddingCache` 已实际实现，采用 LRU 淘汰策略，并提供命中/未命中统计与批量去重能力：

```go
// EmbeddingCache 提供带 LRU 淘汰的嵌入缓存
type EmbeddingCache struct {
    mu       sync.RWMutex
    cache    map[string]*lruEntry // content hash -> vector
    lruList  *list.List
    capacity int

    hits   int64
    misses int64
}

type lruEntry struct {
    key   string
    vec   []float32
    elem  *list.Element
}
```

**核心特性**：

| 特性 | 说明 |
|------|------|
| **LRU 淘汰** | 缓存容量达到上限时，自动淘汰最近最少使用的嵌入向量 |
| **命中统计** | 暴露 `Hits()` / `Misses()` / `HitRate()` 方法，便于监控缓存效率 |
| **批量去重** | `BatchGetOrCompute` 会先对输入文本去重，再批量调用嵌入模型，避免重复计算；返回结果时自动展开还原 |

**使用示例**：

```go
cache := NewEmbeddingCache(1000) // 最多缓存 1000 条

// 单条获取
vec, err := cache.GetOrCompute(ctx, text, embedModel)

// 批量获取（内部自动去重）
vecs, err := cache.BatchGetOrCompute(ctx, texts, embedModel)

// 查看统计
fmt.Printf("hit rate: %.2f%%\n", cache.HitRate()*100)
```
