已实现 (Phase 1+2)：typed Credential + schemas 端点 + 强自动装配（AutoStandardTools、auto TaskStore、WorkspaceManager、Schedule restore、ToolOffload、默认权限）。

此 Studio 使用纯 Go + HTMX 实现（用户选择的轻量路线），无需 Node.js。启动时使用了 NewApp + Auto* 标志，因此本服务器上的 agent 会自动获得丰富工具。

🚀 自动装配 (Auto-Assembly) 效果演示

因为本 Studio 使用了 AutoStandardTools: true + WorkspaceBaseDir + AutoToolOffload + tracingMW (Phase 5) 等：

  • Workspace：每个 session 自动获得隔离的工作目录（文件读写、list、shell 受限）。
  • Standard Tools（自动 + 内部 auto in-memory TaskStore）：
    • 文件：view_text_file, list_directory, write_text_file, edit_text_file, insert_text_file, glob, grep
    • Web：web_fetch
    • JSON：json_parse, json_query
    • Task：TaskCreate / TaskGet / TaskList / TaskUpdate （使用自动创建的内存 TaskStore）
    • Schedule：ScheduleCreate/List/Stop/View （当 ScheduleMgr 可用时）
  • Tool Offload：长耗时工具自动后台化 + Hint 注入恢复上下文。
  • Schedule Restore：持久化 schedule 重启后自动恢复（通过 BTM.Start + loadPersistedSchedules）。
  • Permission：默认 Explore 模式 + 合理规则（写操作 ask，读允许）。
  • Tracing (Phase 5)：TracingMiddlewareAdapter 注入 spans 到 on_reply/on_reasoning 等 hook，可与 OTel/LangSmith 结合（demo 用 NoopTracer，实际用真实 tracer）。

动态创建的 agent/session 会自动获得以上能力。使用下方导航尝试 Agents → 创建 → Chat Demo 来测试自动工具。
