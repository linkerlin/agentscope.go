## 二、v2.1.0 — 工程化加固（技术债清理）

**目标**：修复审计发现的高/中严重度问题，提升代码质量。

### 2.1 Formatter 接口统一 [H2]

```
问题：Anthropic/Gemini Formatter 的方法签名与 Formatter 接口不兼容
方案：
  1. 将 Anthropic/Gemini 的 FormatMessages() 返回类型改为 (any, error)
  2. 将 Gemini 的 FormatContents() 重命名为 FormatMessages()
  3. 补全缺失的 FormatToolChoice() 方法
  4. 验证所有 model 后端通过 Formatter 接口可多态调用
验收：`var _ Formatter = (*AnthropicFormatter)(nil)` 编译通过
```

### 2.2 Agent V1/V2 路径统一 [H3]

```
问题：replyInternal() 与 replyStreamInternal() 大量代码重复
方案：
  1. 提取公共 ReAct 循环逻辑到 `reactLoop()` 函数
  2. Call(V1) 内部包装 ReplyStream(V2) + CollectMessage
  3. 或 Reply(V2) 内部包装 Call(V1) + 事件发射
  4. 保留两个入口但共享核心逻辑
验收：ReActAgent 通过 `var _ agent.V2Agent` 编译断言
```

### 2.3 MemoryType 统一 [M1]

```
方案：
  1. 在 vector/types.go 中定义完整的 MemoryType 常量集
  2. reme_types.go 仅做 `type MemoryType = vector.MemoryType` 别名
  3. 统一为：personal/procedural/tool/profile/summary/identity/history/gene/capsule/evolution_event
验收：两处常量集完全一致
```

### 2.4 安全加固 [M2, M3]

```
方案：
  1. a2a AuthMiddleware 的 JWT 改用 golang-jwt/jwt/v5 库
     - 支持 exp 过期验证
     - 支持 RS256/HS256 算法验证
  2. a2a WebSocket CheckOrigin 改为可配置
     - 默认拒绝跨域
     - 支持 AllowedOrigins 白名单
验收：单元测试覆盖过期 token 拒绝和跨域 WS 拒绝
```

### 2.5 占位桩清理 [M4]

```
方案：
  1. Elasticsearch 向量后端：实现完整 REST API 或标记 Deprecated
  2. Pgvector 向量后端：实现完整 SQL 或标记 Deprecated
  3. 如标记 Deprecated，构造函数返回明确 error 而非静默 no-op
验收：无静默 no-op 操作
```

### 2.6 命名冲突修复 [M6]

```
方案：
  1. react.AgentState → react.ConfigSnapshot（配置快照）
  2. 保留 agent.AgentState（运行时状态）
  3. 更新所有引用
验收：grep 确认无 react.AgentState 残留
```

### 2.7 其他修复

- [L1] Router 移除 circuit-breaker 注释 或 实现简单断路器
- [L2] `rag/InMemoryVectorStore.sortScores` 改用 `sort.Slice`
- [L3] `config.LoadFromFile` 增加 YAML 支持
- [L4] Gemini Dimensions() 改为可配置

- ✅ 压缩水位线追踪（`CompressionWatermark`）— 累积追踪已压缩消息总数

### 3.4 A2A 分布式 ReAct ✅

### 3.5 Evolver 真实 MCP 集成 ✅

## 七、技术决策

### 7.1 保持零 CGO 依赖

ONNX 已采用 HTTP 代理方案。tree-sitter Bash 解析使用 build tag 隔离。未来 WASM 编译需要继续避免 CGO。

### 7.2 V1/V2 兼容策略

V1（Call/CallStream）和 V2（Reply/ReplyStream）共存。不废弃 V1，但 V2 为推荐路径。通过共享核心逻辑减少维护成本。

### 7.3 Mock 先行策略

Evolver、ONNX、向量数据库等外部依赖均采用 Mock/Adapter 先行，生产环境通过 MCP/HTTP 桥接。保持核心包零外部二进制依赖。

### 7.4 文档即代码

所有 API 文档通过 `go doc` 生成。README/docs 中的代码示例必须可编译（通过 examples/ 目录验证）。
