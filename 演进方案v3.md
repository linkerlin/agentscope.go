## 一、Python 参考实现近 2 周的领先点（差距分析）

经逐 PR / 逐文件比对，确认 Python 相对 Go 的**真实领先点**如下（按价值×可行性排序）：

| # | Python 新增（PR / commit） | 能力描述 | Go 现状 | 差距 |
|---|---------------------------|----------|---------|------|
| 1 | `BudgetControlMiddleware` (#1738) | 按 reply 计的加权 token 预算：累计 `in_w*input + out_w*output`，超限注入 HintBlock + 强制 `tool_choice=none`，状态存 `middle_context` 以跨 HITL 恢复 | `middleware/` 仅有 `LoggingMiddleware` 一个参考实现 | **缺失** |
| 2 | `mem0` 长期记忆中间件 (#1775) | 三模式（`static_control` / `agent_control` / `both`）：回复前自动检索、回复后回写；或暴露 `search_memory`/`add_memory` 工具 | Go 有更强的 ReMe 记忆，但**无统一的长期记忆中间件抽象** | **缺失（抽象层）** |
| 3 | TTS 模块 + DashScope 后端 (#1832) | 独立 `tts/` 包：`TTSModelBase`（非实时 `synthesize` + 实时 `push`/`connect`/`close`）、DashScope 后端、实时模型、YAML ModelCard | Go 仅有 `model/openai_tts.go` + `model/audio.go` 接口 + 同步网关端点；**无独立 tts 包、无 DashScope、无实时 TTS** | **缺失** |
| 4 | `TTSMiddleware` (#1832) | 拦截推理文本块，合成语音并以 `DATA_BLOCK_*` 事件注入回复流（实时按 delta `push`，非实时在 `TextBlockEnd` `synthesize`） | **缺失** | **缺失** |
| 5 | Omni 实时音频字幕流 (#58c24fa7) | `_utils/_audio.py`、`streamingAudio.ts`、`AudioContext`，formatter 处理音频多模态 | Go `model.StreamAudio` 是桩（返回 not supported）；网关无实时字幕 SSE/WS；仅有 ModelCard `qwen3.5-omni-plus`（无实现） | **缺失（实现）** |
| 6 | Embedding 按 provider 重构 + ModelCard (#1852) | 拆分为 `_dashscope/`/`_gemini/`/`_ollama/`/`_openai/` 子目录 + 各自 YAML ModelCard | Go `embedding/` 为扁平文件、**无 embedding ModelCard**（仅 chat 模型有 cards） | **部分** |
| 7 | Agent Team / 自定义 Agent 类 (#1833, #1838) | agent service 支持自定义 agent 类、自定义 subagent 模板（leader 从模板生成子 agent） | 网关 `agent_factory` 恒构建 `ReActAgent`；无 agent_class/模板字段 | **缺失** |
| 8 | 权限上下文从 leader 继承 (#1815) | subagent 继承 team leader 的权限上下文 | Go `SubagentTool` 仅传递深度计数器，**不继承权限** | **缺失** |
| 9 | workspace root 注入权限上下文 (#1823) | workspace 根路径加入权限引擎 `Context`，使 ACCEPT_EDITS 能在会话工作区内自动放行 | Go **两侧管道都存在但未连接**：`gateway/agent_session.go:84` 用 `NewEngine`（`WorkingDirs` 为空），`permission.NewEngineWithContext`+`WithWorkingDirs` 仅在单测中用过 | **管道存在、未接线** |
| 10 | 后台任务管理器多进程/分布式重构 (#1849) | message_bus / bg_task 重构以支持多进程与分布式部署 | Go 有 a2a Redis 注册中心 + Schedule，但 bg_task 的多进程解耦形态不同 | **形态不同** |

**Go 已覆盖 / 既有优势（保留并发扬）**：
- Bash 工具 `cwd`：Go 的 `tool/shell` 已支持 `working_dir`（等价）✓
- Glob 跨平台：Go `filepath` 原生处理 ✓
- 事件 metadata：Go 事件系统丰富 ✓
- ReMe 记忆（混合检索 / 知识图谱 / Dream 演化）、a2a 协议（Go 领先 PyV2）、gateway AG-UI / offload、ONNX 本地多模态 —— 均为 Go 独有优势

## 二、实施计划（分 Tier，由高价值高可行性到侵入性递增）

### Tier 1 —— 自包含、高价值、低风险（本次优先实施）

#### 1A. workspace root → 权限上下文接线（对齐 #1823）
- **改动**：`gateway/agent_session.go:84`，将 `permission.NewEngine(mode, defaultWorkspacePermRules())` 改为
  `permission.NewEngineWithContext(permission.NewContext(mode).WithWorkingDirs(wsDir), defaultWorkspacePermRules())`。
- **效果**：ACCEPT_EDITS 模式可在会话工作区内自动放行写操作（修复"管道存在但未接线"的真实缺陷）。
- **测试**：新增 gateway 单测验证 `WorkingDirs` 含 `wsDir`。

#### 1B. `BudgetControlMiddleware`（对齐 #1738）
- **新文件**：`middleware/budget.go` + `middleware/budget_test.go`
- **设计**（地道 Go，复用已确认的事实：reply / reasoning / model_call 三链共享同一 `context.Context`）：
  - 预算累加器经 `context.Context` 携带（`ctxKey{}` + `*budgetAccum{used float64}`），`OnReply` 注入、`OnModelCall` 累加、`OnReasoning` 读取判定。
  - token 用量来源：优先 `msg.Usage`（`message.TokenUsage`，非流式），回退 `msg.Metadata["usage"]`（`model.ChatUsage`，流式）。
  - 超限时：`OnReasoning` 往 `input.Messages` 末尾追加含 HintBlock 的 assistant 消息，并把 `input.ChatOpts` 的 `ToolChoice` 置为 `none`（通过既有 `model` option）。
  - 实例无状态（累加器在 ctx 内），可安全跨 agent 共享。
- **测试**：构造 mock model 返回固定 usage，验证累计、超限注入 hint、强制 none、阈值边界。

#### 1C. `tts/` 包 + `TTSMiddleware`（对齐 #1832）
- **新包** `tts/`：
  - `tts/base.go`：`Model` 接口（`Synthesize(ctx, text, opts) (Response, error)` + 实时 `Connect/Push/Close`）、`Response`（音频块 + `IsLast` + `MediaType`）、`Options`、`Parameters`。
  - `tts/dashscope/`：DashScope TTS 后端（非实时 HTTP `synthesize` + 可选实时 WebSocket `push`）。
  - `tts/openai/`：包装既有 `model.OpenAITTS` 为 `tts.Model`（复用，零重复）。
  - `tts/modelcard.go` + `tts/cards/*.yaml`：TTS ModelCard（qwen3-tts-flash / qwen3-tts-flash-realtime / tts-1 / tts-1-hd）。
- **新文件** `middleware/tts.go` + 测试：拦截推理文本 → 合成 → 经 `hook` 事件 / 网关 AG-UI data block 注入音频。Go 无 Python 的 `DATA_BLOCK_*` 事件，改用既有 `hook.Event` 体系 + 网关 SSE 扩展（在 reply msg 的 `Metadata["audio"]` 携带 base64 PCM 块，或经 stream hook 发射自定义音频事件）。
- **网关**：`gateway/audio_handlers.go` 增加可选 `POST /v1/audio/speech/stream`（SSE 流式合成）。
- **测试**：mock TTS 模型验证中间件注入；DashScope 后端用 httptest mock。

### Tier 2 —— 中等价值 / 适配既有优势

#### 2D. 长期记忆中间件（对齐 #1775，但桥接 Go 优势）
- **新文件** `middleware/longterm_memory.go`：定义 `LongTermMemory` 接口（`Search(ctx, query, opts) ([]Memory, error)` + `Add(ctx, texts, opts) error`）。
- **后端适配器**：
  - `memory.ReMeAdapter`：桥接既有 ReMe 向量记忆（Go 优势，零新依赖）。
  - `memory.Mem0HTTPAdapter`（可选）：通过 HTTP 调用外部 mem0 / Platform，不强依赖 Python mem0 库。
- **三模式**：`static_control`（reply 前注入检索结果 hint）/ `agent_control`（暴露 `search_memory`/`add_memory` 工具）/ `both`。
- **工具**：`tool.MemorySearchTool` / `tool.MemoryAddTool`（实现 `tool.Tool`，经 `toolkit` 注册）。
- **测试**：mock 后端验证三模式 + 工具调用。

#### 2E. Embedding ModelCard 化（对齐 #1852）
- 为 `embedding/` 各 provider 新增 `embedding/cards/*.yaml`（text-embedding-3-large/small、gemini-embedding-001、text-embedding-v3/v4、tongyi-embedding-* 等）。
- 新增 `embedding.ListModelCards()` + 测试（复用 `model/model_card.go` 的 YAML 加载逻辑）。
- 价值：与 chat 模型卡片体系一致，支撑 Studio 动态表单。

### Tier 3 —— 侵入性较高（谨慎、增量）

#### 3F. 自定义 Agent 类 + Subagent 模板（对齐 #1833, #1838, #1815）
- **service 层**：`service/entities.go` 的 `AgentConfig` 增加 `AgentClass string`（默认 `"react"`）+ `SubagentTemplates []SubagentTemplate`。
- **网关**：`gateway/service_handlers.go` 的 `createAgentRequest` 增加同名字段；`agent_factory.go` 增加按 `AgentClass` 的工厂分发（`react` / `router` / `pipeline` / 自定义注册）。
- **subagent 模板**：leader agent 创建时，按模板经 `agent/subagent.go` 的 `AgentProvider` 工厂注册为 subagent 工具；**模板构建时继承 leader 的 `PermissionEngine`**（对齐 #1815）。
- **AgentFactory 注册表**：`gateway/agent_registry.go` 允许外部注册自定义 agent 构造器（`RegisterAgentClass(name, Builder)`）。
- **测试**：多类 agent 构建、模板 subagent 权限继承、网关 CRUD 往返。

### Tier 4 —— 后续对齐（记录，择期）
- Omni 实时音频字幕流（#58c24fa7）：实现 `model.StreamAudio` 真后端 + 网关实时 SSE/WS。
- 后台任务多进程/分布式（#1849）：与 a2a Redis 注册中心统一为分布式任务总线。
- 文档 / CHANGELOG / AGENTS.md / 迁移指南同步。
